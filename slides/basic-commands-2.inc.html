Basic Commands (continued)

Making changes

git add filename.txt # Add specific file
git add . # Add all changes
git commit -m "Fix bug in sensor reading"

Commit messages should be descriptive and start with a verb

Syncing with remote

git pull # Get latest changes
git push # Upload your changes

Viewing history

git log --oneline # See commit history
git diff # See unstaged changes

⚠️ Important Tips

  • Always pull before pushing - get latest changes first
  • Commit often - small, focused commits
  • Write good commit messages - future self will thank you
  • Don't commit secrets - API keys, passwords, etc.
Good commit message:
"Add temperature validation to sensor module"

Bad commit message:
"fixed stuff"
