PR Best Practices

📋 Include in PR Body

  • Unit test results - show tests pass
  • Code coverage metrics - before/after %
  • Manual testing steps - how you verified
  • Screenshots/videos - for UI changes
  • Performance impact - if relevant

👥 Review Requirements

  • 2 reviews minimum - from any team members
  • 1 review from key context holder - familiar with area
  • All reviewers must approve - no merge until all ✅
  • Address all feedback - respond to every comment
  • Re-request reviews - after making changes

⚠️ Important Reminder

Always address feedback, even if a reviewer approves!

An approval doesn't mean you should ignore suggestions. Respond to all feedback and implement reasonable suggestions.

🚫 Common Mistakes: Submitting untested code • Making unrelated changes • Ignoring feedback • Force-pushing after reviews
