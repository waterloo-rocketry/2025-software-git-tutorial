What is Git?

  • Version Control System (VCS) - tracks changes to files over time
  • Distributed - every developer has a complete copy of the project history
  • Collaborative - multiple people can work on the same project simultaneously
  • Branching & Merging - work on features in isolation, then combine them

Why use Git?

  • Never lose work - complete history is preserved
  • See exactly what changed and when
  • Collaborate without stepping on each other's toes
  • Industry standard - used everywhere in software

Git vs GitHub

Git: The version control tool that runs locally on your computer

GitHub: A cloud service that hosts Git repositories and adds collaboration features like Issues and Pull Requests

Think of Git as the engine, and GitHub as the social network built around it.
