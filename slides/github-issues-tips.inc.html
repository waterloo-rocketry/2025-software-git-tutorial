GitHub Issues Tips

✅ Do

  • Search existing issues first
  • Use clear, descriptive titles
  • Include reproduction steps
  • Add relevant labels
  • Link related issues/PRs
  • Update issues as you learn more

❌ Don't

  • Create duplicate issues
  • Use vague titles like "It's broken"
  • Assign yourself unless actively working
  • Leave issues open indefinitely
  • Forget to close when resolved

🎯 Assignment Policy

Only assign yourself to an issue if you're actively working on it

This helps other team members know what's available. If you step away, unassign yourself so others can pick it up.
